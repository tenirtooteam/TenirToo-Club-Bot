# Walkthrough — Двухуровневая архитектура документации (ядро + OKF-бандл) + graphify

Отчёт о реализации фичи `001-okf-docs-graphify`. Исполнитель: Claude Opus через `speckit-implement`.

## Changes made (Внесённые изменения)

**Новый OKF-бандл `docs/knowledge/` (отслеживается в Git):**
- `index.md` — точка входа с прогрессивным раскрытием (единственный файл бандла в pre-read Route A).
- `log.md` — хронология изменений бандла.
- `db-schema.md` — полный DDL, перенесён из `[PL-3.1]`.
- `module-registry.md` — реестр модулей и инвентарь функций, перенесён из `[PL-2.2]`.
- `features/design-system.md` — дизайн-токены Mini App, перенесены из `[CP-4]`.

**Тонкое ядро (правила остаются в гарантированном контексте):**
- `PROJECT_LOGIC.md` — секции `[PL-2.2]` и `[PL-3.1]` заменены на стабы-указатели с сохранением якорей; императив `[PL-2.2.50]` и факт `[PL-3.1.1]` оставлены в ядре. Размер 51.2 → 34.4 КБ.
- `CONTEXT_PROMPT.md` — починено повреждение на границе `[CP-2]/[CP-3]`; восстановлен самостоятельный заголовок `## [CP-3]`; `[CP-3.6]`/`[CP-3.7]` дедуплицированы ссылками на `[PL-4.5]`/`[PL-6.12]` и `[PL-6.1]`/`[PL-6.2]`/`[PL-6.18]`; список фич `[CP-2]` сжат до одной строки на фичу; `[CP-4]` вынесён в бандл. Размер 38.6 → 33.8 КБ.

**Синхронизация процесса:**
- `GEMINI.md` — pre-read Route A теперь включает `docs/knowledge/index.md`; добавлены строки в § FILE REGISTRY и § CONTENT OWNERSHIP RULES; онбординг указывает на graphify при наличии `graphify-out/`.
- `docs-update/SKILL.md` — CMD-1/CMD-2 маршрутизируют описательный контент в бандл с атомарностью (concept + index + log) и producer-контрактом.
- `.gitignore` — добавлен `graphify-out/`.

**Валидация и граф:**
- `tests/test_knowledge_bundle.py` — новый набор из 6 контрактных тестов; `tests/fixtures/pl_anchors_baseline.txt` — заморожённый инвентарь 250 якорей.
- `graphify-out/` (git-ignored) — граф знаний: 1002 узла, 2309 рёбер, 66 сообществ.

Исходный код бота (`handlers/`, `services/`, `database/`, `keyboards/`, `middlewares/`, `web/`) **не изменялся**.

## What was tested (Что тестировалось)

- **TDD-порядок**: набор `test_knowledge_bundle.py` написан и зафиксирован красным ДО создания бандла (`test_cp_corruption_absent` падал на живом повреждении, `test_pl_anchors_preserved` был зелёным до-миграционно).
- **6 контрактных тестов**: front matter, консистентность index↔файлы, сохранность 250 якорей в union(ядро+бандл), отсутствие висячих ссылок, отсутствие повреждения, непустой log.
- **Мутационные проверки (SC-003)**: удаление `type:`, удаление записи index, переименование core-only якоря `[PL-6.14]` — каждая корректно роняет набор; после восстановления — зелёный.
- **Полная регрессия**: `python -m pytest` — 105 passed, 1 skipped, 0 failed; `git status tests/` показывает только новые файлы.
- **Проектный ruff-гейт**: поймал неиспользуемый импорт в новом тесте (исправлено).
- **graphify query (SC-006)**: архитектурный вопрос о зависимостях от фасада БД отвечен из графа с указанием `source_location` без открытия исходников.

## Validation results (Результаты валидации)

- **SC-001**: pre-read 89.8 → 69.5 КБ (**−23%**). Обоснованный частичный результат: по решению Шэфа `> Rationale:`-строки правил CP-3 оставлены в ядре (объяснения предотвращают нарушение правил) — цель −40% сознательно не форсировалась ради сохранности правил.
- **SC-002**: ни один императив не удалён; перенесён только описательный контент. Все **250** якорей `PL-x.y` сохранены (тест зелёный).
- **SC-003**: три класса деградации бандла доказанно ловятся.
- **SC-004**: существующий набор зелёный, правок существующих тестов ноль.
- **SC-005**: висячих ссылок `PL-x.y` ноль.
- **SC-006**: архитектурный вопрос отвечен из графа graphify.

**Известное ограничение графа**: 265 висячих рёбер (расхождение ID между семантическими субагентами и AST) — типичный артефакт, не фатально, граф построен и используется. Отражено в `baseline.md`.
