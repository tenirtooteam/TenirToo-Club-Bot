# Contract: `SQLiteStorage` (feature 012)

**Дата**: 2026-07-17 | **Модель данных**: [data-model.md](../data-model.md) |
**Решения**: [research.md](../research.md)

Контракт двусторонний: сверху — то, что требует aiogram 3.4.1 от любого `BaseStorage`; снизу —
то, что storage обещает базе и проекту. Обе стороны верифицированы по исходникам в `venv/`.

## 1. Верхняя граница — контракт aiogram

`class SQLiteStorage(BaseStorage)` реализует все пять абстрактных методов
`[VERIFIED: aiogram/fsm/storage/base.py]`.

### `async set_state(key: StorageKey, state: StateType = None) -> None`

- Принимает `State` | `str` | `None`. Разворачивание — как в эталоне:
  `state.state if isinstance(state, State) else state` `[VERIFIED: memory.py:41]`.
- Пишет `state`, не трогая `data`.
- После записи применяет порог удаления I-4.

### `async get_state(key: StorageKey) -> Optional[str]`

- Возвращает имя состояния либо `None`.
- Неизвестный ключ → `None` (I-2), без исключения.

### `async set_data(key: StorageKey, data: Dict[str, Any]) -> None`

- **Замена**, не слияние.
- Сериализация — JSON (D-3). Несериализуемое значение → `TypeError` **всплывает** (D-3): молча
  потерянное состояние хуже громкой ошибки.
- После записи применяет порог удаления I-4.

### `async get_data(key: StorageKey) -> Dict[str, Any]`

- Неизвестный ключ → `{}` (I-2), без исключения.
- Нечитаемый JSON → `{}` + `logger.warning` (FR-009), без исключения.
- Возвращает **копию**: мутация возвращённого словаря не должна влиять на хранилище
  (`MemoryStorage` отдаёт `.copy()` `[VERIFIED: memory.py:50]`; у нас это выходит само —
  словарь каждый раз собирается из JSON).

### `async update_data(key, data) -> Dict[str, Any]`

- **Не переопределяется** (D-6): унаследованная реализация `get_data` → `update` → `set_data`
  даёт точное совпадение с эталоном.

### `async close() -> None`

- **No-op** (D-5). Вызывается диспетчером на остановке
  `[VERIFIED: fsm/middleware.py:110]`; соединением владеет `database/connection.py`, и storage
  не имеет права его закрывать.

## 2. Нижняя граница — контракт с базой и проектом

- **Соединение**: только через `get_conn()` из `database/connection.py` — общее
  переиспользуемое соединение + WAL, паттерн фичи 008 №15 (FR-006). Своего соединения storage
  не открывает и не закрывает.
- **Запись**: `INSERT ... ON CONFLICT (PK) DO UPDATE` внутри `with conn:` (commit/rollback —
  ответственность вызывающего, как во всём проекте). UPSERT доступен
  `[VERIFIED: sqlite 3.45.1 ≫ 3.24]`.
- **Конструктор не трогает БД** (D-8): ни `CREATE TABLE`, ни `get_conn()`. `loader.py` создаёт
  storage на импорте, а `init_db()` отрабатывает позже, внутри `main()`
  `[VERIFIED: loader.py, main.py:49]`.
- **Схема** — приватная деталь (I-5): через фасад `database/db.py` ре-экспортируется только
  класс, ни одной доменной операции над FSM у фасада не появляется.
- **Синхронный `sqlite3` внутри `async`** — принято сознательно: ровно то же делает весь
  `database/*` со времён 008 №15. Async-драйвер вне scope (спека, Assumptions).

## 3. Проводка

~~~
# loader.py
from database.db import SQLiteStorage      # через фасад — R-ARCH-1 (D-1)
dp = Dispatcher(storage=SQLiteStorage())   # было: MemoryStorage()
~~~

Больше в проводке не меняется ничего. Потребители (`handlers/*`, `services/ui_service.py`) не
трогаются вовсе — SC-006 требует ноль правок в местах вызова FSM.

## 4. Что контракт обещает потребителю

| Обещание | Требование | Проверка |
|---|---|---|
| Состояние и данные переживают рестарт | FR-001 | новый экземпляр storage поверх той же БД читает записанное |
| Владельцы изолированы | FR-002, I-6 | запись под разными ключами не пересекается |
| Storage безразличен к смыслу ключей | FR-003 | произвольный набор ключей round-trip'ится |
| Типы сохраняются | FR-004, I-3 | `list[int]`/`bool`/`None` возвращаются собой |
| Потребители не меняются | FR-005, SC-006 | существующий корпус тестов зелёный без правок |
| Трекинг-ключи не гибнут при снятии состояния | `R-FSM-1`, I-4 | `set_state(None)` при непустых данных строку **не** удаляет |
| Битая запись не роняет бота | FR-009 | подложить мусор в `data` → `{}` + лог, без исключения |
| Пустые записи не копятся | FR-010, I-4 | `state=None` + `data={}` → строки нет |
| Один владелец — одна строка | I-1 | две записи при `thread_id=None` → `count(*) == 1` |
