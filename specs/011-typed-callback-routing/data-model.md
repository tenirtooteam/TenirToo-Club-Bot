# Data Model: Типизированный роутинг колбэков

**Feature**: 011-typed-callback-routing | **Date**: 2026-07-14 | **Phase**: 1

Сущности фичи — не данные в БД. Схема БД, фасад `database.db` и контракт мутаций не затрагиваются (FR-014). Модель описывает **формат маршрута** как объект первого класса.

---

## Сущность 1: Объявление маршрута (`CallbackData`-фабрика)

Единственный источник правды об одном маршруте (FR-001). Живёт в новом модуле `callbacks.py` (репозиторный корень, рядом с `keyboards/`, `handlers/`, `services/`).

**Атрибуты**:

| Атрибут | Смысл | Ограничение |
|---|---|---|
| `__prefix__` | Уникальное имя маршрута | Не содержит `:`; уникален в реестре (проверяется тестом) |
| `__separator__` | Разделитель | `:` (умолчание aiogram); не переопределяем |
| поля модели | Именованные типизированные параметры | Только `int`/`str`/`bool`/`Enum` — ограничение `_encode_value`; значение не содержит `:` |

**Правила валидации** (обеспечиваются библиотекой, `research.md` §1):
- `pack()` → `ValueError`, если результат >64 байт;
- `unpack()` → `TypeError` при неверной арности, `ValueError` при неверном префиксе, `ValidationError` (⊂ `ValueError`) при непроходящем типе;
- все поля сериализуются всегда — «короткой формы» не существует (D-2).

**Инвариант**: имя поля — контракт. Позиция поля в строке — деталь сериализации, на которую **никто не имеет права опираться** (FR-003).

## Сущность 2: Реестр маршрутов

Декларативное соответствие «фабрика → экран», заменяет цепочку `if` в `generic_navigator` (FR-006).

**Структура**: `{prefix: (Factory, render_fn)}` — плоский словарь, ключ = `__prefix__`.

**Правила**:
- поиск ключа **точный** (`dict.__getitem__`), не `startswith`, не `in` (FR-002);
- отсутствие ключа → защитный возврат (FR-008);
- `render_fn` принимает распакованный объект и вызывает соответствующий `UIService.*`-экран;
- реестр — единственное место, где маршрут связывается с экраном.

**Инвариант R-1**: каждая фабрика в `callbacks.py` присутствует в реестре ровно один раз (тест).
**Инвариант R-2**: префиксы уникальны (тест) — гарантия невозможности коллизии имён, которую подстрочный матчинг допускал.

## Сущность 3: Постоянный маршрут

Маршрут без параметров. Остаётся простой строковой константой (спека, Assumptions).

**Разграничение с параметризованным** — по наличию `:`:

| Вход навигатора | Трактовка |
|---|---|
| `CallbackData`-объект | Параметризованный, уже разобран |
| `str` без `:` | Постоянный → словарь констант |
| `str` с `:` | Параметризованный → префикс = `split(":")[0]` → реестр → `unpack()` |

**Инвариант C-1**: ни один постоянный маршрут не содержит `:` (тест) — иначе разграничение рушится.

**Состав** (без параметров, не паджинируются): `admin_main`, `user_main`, `user_profile_view`, `roles_dashboard`, `roles_faq`, `moderator`, `templates_faq`, `event_list`, `event_pending_list`, `landing`, `close_menu`, `ignore`, `noop`, `admin_confirm_onboarding`, `add_group_start`, `add_user_start`, `sheets_export_all`, `sheets_import_all`.

## Инвентаризация маршрутов в scope

Семейство легаси-навигатора (D-4). Столбец «Дефект» ссылается на `research.md` §2.

> **Поправка (фаза 4, утверждена Шэфом)**: исходная инвентаризация была неполной. Реальность: **17 паджинируемых маршрутов**, а не 13. Добавлены: `moderator` (был ошибочно записан в постоянные — на деле паджинируется, страница молча игнорировалась), а также `add_topic_to`, `mod_add_user_list`, `search_pg` — они вне семейства навигатора (D-4), но попадают в границы **механически**, как вызывающие общего паджинатора: его контракт сменился, и без `page_cb` их клавиатуры не соберутся. У них мигрирует только вход на экран/страницу; их action-кнопки и разбор в обработчиках не трогаются.
>
> Отдельно: `list_users_roles` — фабрика **без собственного производителя стрелок**. Его экран переиспользует `users_list_kb`, чей паджинатор метит стрелки маршрутом `manage_users`, поэтому переход на стр. 2 уводит на «Клубный реестр» (другой заголовок, тот же список). Поведение существует сегодня, в санкционированный список FR-015…FR-017 не входит — зафиксировано характеризацией, заведено в роадмап.

## Сущность 4: Контракт паджинатора

`build_paginated_menu` (`keyboards/pagination_util.py:37`) — единственный производитель страничных стрелок.

**Было**: `callback_prefix: str` → `f"{callback_prefix}_pg_{page±1}"`.
**Стало**: `page_cb: CallbackData` (с полем `page`) → `page_cb.model_copy(update={"page": page±1}).pack()`.

**Инвариант P-1**: `page_cb` обязан иметь поле `page` (тест).
**Затрагивает**: 16 вызывающих (`admin_kb` ×9, `moderator_kb` ×6, `user_kb` ×1).

**Следствие**: `UIService.PAGINATED_CMDS` (`ui_service.py:13`) — дублирующий реестр имён страничных команд — **удаляется**. Страничность следует из наличия поля `page` у фабрики (FR-005).

**Побочный эффект**: `build_paginated_menu` также сравнивает `s_btn.callback_data == callback_prefix` (`pagination_util.py:91`) для распознавания кнопки «Назад» и парсит `help:`-кнопки (`pagination_util.py:95-99`). Обе ветки опираются на строковый префикс и мигрируют вместе с контрактом.

## Переходы состояний

Фича не вводит состояний. Единая точка сброса FSM при стандартном переходе (`state.set_state(None)` + `clear_fsm_data_safely`, `ui_service.py:280-281`) сохраняется в навигаторе без изменений (FR-007, `R-FSM-1`).
