# Contract: Маршруты колбэков

**Feature**: 011-typed-callback-routing | **Phase**: 1

Внешний интерфейс фичи — контракт между **производителем** кнопки (`keyboards/*`), **транспортом** (Telegram) и **потребителем** (`handlers/*` → `UIService.generic_navigator`). Полный реестр маршрутов и их поля — в [data-model.md](../data-model.md); здесь только контракт, обязательный к соблюдению обеими сторонами.

---

## C-1. Формат провода

```text
<prefix>:<value1>:<value2>:...
```

- разделитель `:` (умолчание aiogram 3, `research.md` §1);
- `<prefix>` уникален в реестре и не содержит `:`;
- значения сериализуются в порядке объявления полей; **порядок — деталь сериализации, не контракт** (FR-003);
- присутствуют **все** поля всегда; короткой формы нет (D-2);
- полная строка ≤ 64 байт (лимит Telegram), обеспечивается `pack()`.

Постоянные маршруты (без параметров) — голая строка без `:` (например, `admin_main`).

---

## C-2. Обязанности производителя (`keyboards/*`)

| MUST | Проверяется |
|---|---|
| Строить данные параметризованного колбэка **только** через `Factory(...).pack()` | FR-004; тест-гейт на отсутствие ручной сборки |
| Не конкатенировать и не интерполировать данные параметризованного колбэка из фрагментов | FR-004 |
| Передавать паджинатору **экземпляр** фабрики с полем `page`, не строковый префикс | FR-005, инвариант P-1 |

**Запрещено**: `f"user_info_{uid}"`, `prefix + "_" + str(x)`, ручная обрезка до 64 байт.
**Причина запрета обрезки**: обрезание строки посередине даёт синтаксически «валидный», но семантически битый маршрут, тихо уходящий в fallback. `pack()` поднимает `ValueError` на этапе сборки клавиатуры — отказ становится громким и попадает в тест, а не к пользователю (FR-011).

---

## C-3. Обязанности потребителя (`handlers/*`)

| MUST | Проверяется |
|---|---|
| Матчить вход через `Factory.filter()`, не через `F.data.startswith(...)` | FR-010 |
| Делегировать переход навигатору, оставаясь тонким | `R-ARCH-3`, FR-006 |
| Синтетические переходы (из хендлеров сообщений) строить объектом фабрики, не f-строкой | FR-003 |

`Factory.filter()` инжектит в хендлер kwarg `callback_data` — уже распакованный объект. Хендлер передаёт его навигатору как есть (D-6).

---

## C-4. Обязанности навигатора (`UIService.generic_navigator`)

Сигнатура: `generic_navigator(state, event, callback_data: str | CallbackData)` — имя параметра сохранено (`R-UI-3`, D-6).

Порядок разрешения:

1. `CallbackData`-объект → использовать напрямую;
2. `str` без `:` → точный поиск в словаре постоянных маршрутов;
3. `str` с `:` → `prefix = value.split(":")[0]` → **точный** поиск в реестре → `Factory.unpack(value)`;
4. не найдено / `unpack()` поднял `(TypeError, ValueError)` → **защитный возврат**.

| MUST | Проверяется |
|---|---|
| Точный поиск маршрута; никаких `in` / `startswith` по имени | FR-002, SC-001 |
| Извлечение параметров по имени поля; никаких `p[-1]` / `p[3]` | FR-003, SC-002 |
| Ловить ровно `(TypeError, ValueError)` вокруг `unpack()` | D-1 |
| Сохранять единую точку сброса FSM при стандартном переходе | FR-007, `R-FSM-1` |
| Все переходы — через `UIService.sterile_show` | FR-009, `R-UI-1` |

**Защитный возврат** (существующее поведение, `ui_service.py:387-388`, сохраняется дословно): `logger.warning` + `show_admin_dashboard(text="⚠️ Ошибка навигации. Возврат в главное меню:")`. Наружу исключение не выходит (FR-008, `R-UI-3`, `R-SEC-2`).

## C-5. Контракт паджинатора

```text
build_paginated_menu(item_buttons, static_buttons, page, limit, page_cb: CallbackData, ...)
```

- `page_cb` MUST иметь поле `page` (инвариант P-1);
- стрелки: `page_cb.model_copy(update={"page": page ± 1}).pack()`;
- `UIService.PAGINATED_CMDS` удаляется: страничность = наличие поля `page` (FR-005).

## C-6. Что контракт НЕ покрывает

| Вне контракта | Где живёт |
|---|---|
| Авторизация и права | `PermissionService`, `R-SEC-3`, `R-ARCH-7` — **не меняются** (FR-013) |
| Доступ к данным | Фасад `database.db`, `R-ARCH-1/2` — не меняется (FR-014) |
| Тексты и состав экранов | Не меняются (FR-014) |
| Семейства `event_*` / `ann_*` / `date_*`, разбор `search_start_`/`search_pick_` | Вне scope (D-4) |

**Явно**: типизация маршрута **не** является проверкой прав. `Factory.filter()` гарантирует, что `topic_id` — целое число, и ничего не говорит о праве пользователя на этот топик. Серверная перепроверка авторитета остаётся обязательной ровно там же, где сегодня (`R-SEC-3`).

## C-7. Совместимость

Колбэки нигде не персистятся — живут только внутри активных сообщений Telegram. Старый формат (`user_info_5`) после миграции:

- фильтры `Factory.filter()` его не примут → сработает глобальный fallback неотвеченных колбэков (`R-SEC-2`);
- если дойдёт до навигатора — не содержит `:` → поиск в константах → не найден → защитный возврат (C-4.4).

Обе ветки безопасны. Обратная совместимость формата **не требуется** (спека, Assumptions).
