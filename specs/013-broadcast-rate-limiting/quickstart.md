# Quickstart / Validation — Broadcast Rate-Limiting (013)

Как убедиться, что фича работает. Реализация — в `tasks.md`; здесь только запуск и ожидаемый исход.

## Prerequisites

- venv проекта: тесты гонять `venv/Scripts/python.exe -m pytest` (глобальный Python без pytest).
- Никакой реальной сети — все Telegram-вызовы мокаются (`mock_bot`, R-TEST-2).

## Репро-тесты (пишутся ПЕРВЫМИ и падают до фикса — R-PROC-3)

Каждый дефект получает падающий тест до правки:

1. **Срез @all на 50** — `test_send_native_all_covers_all_authorized`: топик со 120 авторизованными →
   ожидаем 3 сообщения, все 120 UID упомянуты. До фикса: 1 сообщение, покрыты только первые 50.
2. **Потеря под 429** — `test_send_to_users_survives_flood_wait`: `mock_bot.send_message` бросает
   `TelegramRetryAfter` на 2-й отправке, потом успех → ожидаем, что 2-й адресат в итоге получил
   сообщение и список дошёл до конца. До фикса: исключение проглочено, адресат потерян.
3. **Рост кэша** — `test_alert_cache_bounded`: наполнить `_alert_cache` множеством протухших пар,
   вызвать алерт → ожидаем ограниченный размер. До фикса: размер растёт монотонно.
4. **Ungated @all** — `test_all_mention_requires_moderator`: @all от не-модератора → 0 рассылок;
   `test_all_mention_cooldown` → повтор < cooldown → 0 рассылок. До фикса: рассылка уходит всегда.

## Ожидаемый исход (маппинг на Success Criteria)

- **SC-001**: `send_to_users` под смоделированным flood-wait доставляет всем достижимым адресатам.
- **SC-002/SC-003**: `send_native_all` на 51+ покрывает 100% авторизованных, срез на 50 отсутствует.
- **SC-004/SC-005**: @all от не-модератора и повтор в пределах кулдауна → 0 рассылок; суперадмин — без
  кулдауна.
- **SC-006/SC-007**: размер `_alert_cache` ограничен TTL/потолком; окна дедупа 60 c/3600 c без изменений.
- **SC-008**: полный корпус зелёный без правок существующих тестов рассылок вне scope.

## Живой прокликк (на Шэфе, вне автотестов)

Автотесты доводят до предела без реального Telegram. Ручная проверка:

- **@all модератором** в топике с >50 авторизованными → приходит несколько сообщений, пуш получают все.
- **@all рядовым участником** (write-capable, но не модератор) → сообщение-триггер исчезает, рассылки
  нет, в топике тихо.
- **@all дважды подряд модератором** → второй раз в пределах минуты рассылки нет.
- **@all суперадмином дважды подряд** → обе рассылки уходят (кулдаун не действует).
- **Массовая ЛС** (аудит-заявка на большом списке) → доходит до всех, лог без обрывов на 429.
