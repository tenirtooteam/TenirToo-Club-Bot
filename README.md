# Telegram Bot: Access Control and Community Management

Профессиональный модульный Telegram-бот на базе **aiogram 3**, разработанный для туристического клуба **«Теңир-Тоо»**. Система обеспечивает надежное управление доступом к темам форума и автоматизацию учета участников.

## 🏗 Архитектура системы

Проект следует принципам слоистой архитектуры (Layered Architecture) с использованием паттерна **Фасад** для работы с данными.



### Схема взаимодействия:
1. **User/Admin** посылает сигнал через Telegram API.
2. **Middlewares** выполняют первичную фильтрацию: авто-регистрацию новичков и "стелс-модерацию" сообщений в топиках.
3. **Handlers** определяют бизнес-логику (обработка команд, переходы по меню).
4. **UIService** управляет чистотой интерфейса: удаляет старые меню и команды, предотвращая "вереницу" сообщений в чате.
5. **Database Layer** (Facade) обеспечивает атомарный доступ к SQLite через транзакции и WAL-режим.

### Структура директорий:
* **main.py** — Точка входа, конфигурация логов и роутеров.
* **loader.py** — Инициализация Bot и Dispatcher.
* **database/** — Уровень данных (SQLite + WAL mode).
    * **connection.py** — Настройка БД и индексов.
    * **members.py** / **access.py** — Транзакционное управление данными.
    * **db.py** — Единый фасад доступа.
* **services/** — Бизнес-логика (UI управление, очистка чата).
* **handlers/** — Модульные роутеры (admin, user, common).
* **middlewares/** — access_check.py: Модерация, авто-регистрация и синхронизация имен.
* **keyboards/** — Генерация Inline-меню (Admin & User).
* **logs/** — Журналы системы с ротацией (5MB).

## 🚀 Примеры использования

### Для Администратора:
1. **Настройка доступа**: Введите `/admin` -> `📂 Группы доступа` -> `➕ Создать группу`. После создания выберите группу и добавьте в неё ID топиков форума через кнопку `📍 Список топиков`.
2. **Управление пользователями**: Перейдите в `👥 Пользователи` -> `user_name` -> `🔐 Управление группами`. Здесь можно в один клик выдать или отозвать права доступа к наборам топиков.
3. **Синхронизация имен**: При изменении имени пользователя или названия топика в интерфейсе админа, изменения автоматически улетят в БД и API Telegram.

### Для Участника:
1. **Личный кабинет**: Команда `/start` открывает меню, где кнопка `👤 Мой профиль` показывает текущие группы доступа.
2. **Проверка прав**: В разделе `📍 Мои топики` участник видит весь список тем форума. Значок ✅ означает право на отправку сообщений, ❌ — топик доступен только для чтения.

## 🛠 Troubleshooting (Решение проблем)

| Проблема | Причина | Решение |
| :--- | :--- | :--- |
| **sqlite3.OperationalError: database is locked** | Конкурентная запись в БД. | Проверьте, что в `connection.py` включен WAL-режим. |
| **Кнопка "Закрыть" не работает** | Потерян `last_menu_id` в FSM. | Убедитесь, что в хендлерах вместо `state.clear()` используется `state.set_state(None)`. |
| **Бот не удаляет сообщения в группе** | Недостаток прав или тип чата. | Бот должен быть администратором группы с правом "Удаление сообщений". |
| **ValueError: BOT_TOKEN is not set** | Ошибка загрузки `.env`. | Убедитесь, что файл `.env` находится в корне и переменные названы корректно. |

## Установка

1. Настройте **.env** по примеру **.env.example**.
2. Установите зависимости: `pip install -r requirements.txt`.
3. Запуск: `python main.py`.
