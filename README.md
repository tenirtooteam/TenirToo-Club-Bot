# Telegram Bot: Access Control and Community Management

## Обзор
Профессиональный модульный Telegram-бот на базе **aiogram 3**, разработанный для туристического клуба **«Теңир-Тоо»**. Система обеспечивает надёжное управление доступом к темам (топикам) форума внутри Telegram-супергруппы, автоматизацию учёта участников клуба и защиту интерфейса от ошибок двойных нажатий.

---

## Архитектура
Проект следует принципам слоистой архитектуры (Layered Architecture) с использованием паттерна **Фасад** для работы с данными.

### Схема взаимодействия:
1. **User/Admin** посылает сигнал через Telegram API.
2. **Middlewares** выполняют первичную фильтрацию: авто-регистрацию пользователей и стелс-модерацию сообщений в топиках.
3. **Handlers** определяют бизнес-логику (обработка команд, переходы по меню).
4. **Services** управляют чистотой интерфейса и защищают от спама нажатий.
5. **Database Layer (Facade)** обеспечивает атомарный доступ к SQLite через транзакции и WAL-режим.

## Ключевые возможности
1. **Стелс-модерация**: топики закрыты по умолчанию — бот беззвучно удаляет сообщения пользователей без прав доступа.
2. **Гибкое управление доступом**: администратор создаёт группы доступа, привязывает к ним топики и пользователей. Реализовано полное каскадное удаление «веток-призраков» из БД.
3. **Защищённый UI**: интерфейс не выдаёт ошибок при двойном клике, старые inline-клавиатуры уничтожаются автоматически.
4. **Синхронизация имён топиков**: переименование топика в Telegram автоматически отражается в локальной БД (Telegram → БД, через `ForumUtilityMiddleware`).
5. **Авто-регистрация участников**: каждый пользователь автоматически вносится в базу данных при первом контакте с ботом, без участия администратора.

| Файл | Назначение |

## 🛠 Troubleshooting

| Проблема | Причина | Решение |
| :--- | :--- | :--- |
| **sqlite3.OperationalError: database is locked** | Конкурентная запись в БД | Проверьте, что в `connection.py` включён WAL-режим |
| **Кнопка "Закрыть" не работает** | Потерян `last_menu_id` в FSM | Убедитесь, что используется `state.set_state(None)` |
| **Бот не удаляет сообщения в группе** | Недостаток прав | Бот должен быть администратором с правом «Удаление сообщений» |
| **Топик удалён в Telegram, но висит в БД** | Несовпадение жизненного цикла | Используйте кнопку «Удалить из БД» в меню топиков администратора |

## Настройка и запуск
1. Переименуйте `.env.example` в `.env` и заполните переменные (`BOT_TOKEN`, `ADMIN_ID`, `GROUP_ID`).
2. Установите зависимости: `pip install -r requirements.txt`.
3. Запустите бота: `python main.py`.
