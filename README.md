# TITLE: Telegram Bot: Access Control and Community Management

## OVERVIEW
Профессиональный модульный Telegram-бот на базе **aiogram 3**, разработанный для туристического клуба **«Теңир-Тоо»**. Система обеспечивает надежное управление доступом к темам (топикам) форума внутри Telegram-супергруппы, автоматизацию учета участников клуба и защиту интерфейса от ошибок двойных нажатий.

## ARCHITECTURE
Проект следует принципам слоистой архитектуры (Layered Architecture) с использованием паттерна **Фасад** для работы с данными.

### Схема взаимодействия:
1. **User/Admin** посылает сигнал через Telegram API.
2. **Middlewares** выполняют первичную фильтрацию: авто-регистрацию новичков и "стелс-модерацию" сообщений в топиках.
3. **Handlers** определяют бизнес-логику (обработка команд, переходы по меню).
4. **Services (UIService, CallbackGuard)**: управляют чистотой интерфейса (удаляют старые меню и команды) и защищают от спама нажатий, предотвращая краш интерфейса.
5. **Database Layer (Facade)**: обеспечивает атомарный доступ к SQLite через транзакции и WAL-режим.

### Структура директорий:
- **main.py** — Точка входа, конфигурация логов и роутеров.
- **loader.py** — Инициализация Bot и Dispatcher.
- **database/** — Уровень данных (SQLite + WAL mode).
    - **connection.py** — Настройка БД и индексов.
    - **members.py** / **access.py** — Транзакционное управление данными (вкл. полное удаление топиков).
    - **db.py** — Единый фасад доступа.
- **services/** — Бизнес-логика.
    - **ui_service.py** — Управление UI окнами.
    - **callback_guard.py** — Защита инлайн-кнопок от двойных нажатий.
- **handlers/** — Модульные роутеры (admin, user, common).
- **middlewares/** — access_check.py: Модерация, авто-регистрация и синхронизация имен.
- **keyboards/** — Генерация Inline-меню (admin_kb, user_kb).
- **logs/** — Журналы системы с ротацией (5MB).

## KEY FEATURES
1. **Стелс-модерация**: Топики закрыты по умолчанию. Бот беззвучно удаляет сообщения пользователей, не имеющих прав доступа.
2. **Изолированное управление топиками**: Администратор может создавать группы доступа, добавлять в них топики и привязывать пользователей. Реализовано полное каскадное удаление "веток-призраков" из БД.
3. **Защищенный UI**: Интерфейс не выдает ошибок при двойном клике благодаря декораторам, а старые инлайн-клавиатуры автоматически уничтожаются для чистоты истории чата.
4. **Синхронизация имен**: При изменении имени в боте, изменения автоматически дублируются в API Telegram.

## 🛠 Troubleshooting (Решение проблем)
| Проблема | Причина | Решение |
| :--- | :--- | :--- |
| **sqlite3.OperationalError: database is locked** | Конкурентная запись в БД. | Проверьте, что в `connection.py` включен WAL-режим. |
| **Кнопка "Закрыть" не работает** | Потерян `last_menu_id` в FSM. | Убедитесь, что используется `state.set_state(None)`. |
| **Бот не удаляет сообщения в группе** | Недостаток прав или тип чата. | Бот должен быть администратором с правом "Удаление сообщений". |
| **Топик удален в Telegram, но висит в БД** | Несовпадение жизненного цикла. | Воспользуйтесь кнопкой "Удалить из БД" в меню всех топиков администратора. |

## CONFIGURATION
1. Переименуйте `.env.example` в `.env` и настройте переменные среды (BOT_TOKEN, ADMIN_ID, GROUP_ID).
2. Установите зависимости проекта: `pip install -r requirements.txt`.
3. Запустите бота командой: `python main.py`.
