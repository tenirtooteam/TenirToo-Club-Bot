# Telegram Bot: Access Control and Community Management

## Обзор
Профессиональный модульный Telegram-бот на базе **aiogram 3**, разработанный для туристического клуба **«Теңир-Тоо»**. Система обеспечивает надёжное управление доступом к темам (топикам) форума внутри Telegram-супергруппы, автоматизацию учёта участников клуба и защиту интерфейса от ошибок двойных нажатий.

---

## Архитектура
Проект следует принципам слоистой архитектуры (Layered Architecture) с использованием паттерна **Фасад** для работы с данными.

### Схема взаимодействия:
1. **User/Admin** посылает сигнал через Telegram API.
2. **Middlewares** выполняют первичную фильтрацию: авто-регистрацию пользователей и стелс-модерацию сообщений в топиках.
3. **Handlers** определяют бизнес-логику (обработка команд, переходы по меню).
4. **Services** управляют чистотой интерфейса и защищают от спама нажатий.
5. **Database Layer (Facade)** обеспечивает атомарный доступ к SQLite через транзакции и WAL-режим.

## Ключевые возможности
1. **Стелс-модерация**: топики закрыты по умолчанию — бот беззвучно удаляет сообщения пользователей без прав доступа.
2. **Шаблонное управление доступом**: Доступ управляется исключительно через прямые записи (`direct_topic_access`). Глобальные группы (Templates) служат инструментом для массового копирования прав, что полностью изолирует runtime-доступ от членства в группах.
3. **Декларативный Стерильный UI**: декоратор `@UIService.sterile_command` и централизованный навигатор `generic_navigator` позволяют управлять всем интерфейсом бота из одной точки. Временные сообщения (ошибки, промпты) отслеживаются через **стек очистки** (`last_menu_ids`) и удаляются автоматически, гарантируя полное отсутствие «мусора» в чате.
4. **Синхронизация имён топиков**: переименование топика в Telegram автоматически отражается в локальной БД (Telegram → БД, через `ForumUtilityMiddleware`).
6. **Дашборд ролей и FAQ**: общая панель «Роли» для администраторов и модераторов с подробным справочником прав и списком всех активных назначений.
7. **Контекстное управление ролями**: назначение прав перенесено напрямую в карточку пользователя, что исключает лишние шаги поиска при администрировании.
8. **Единый интерфейс поиска**: при выборе пользователя/группы/топика в любом меню — всегда отображается список (≤7 записей) и кнопка ручного поиска (если записей больше). Поиск поддерживает все типы сущностей и обрабатывается через `_fetch_search_results` в `handlers/common.py`.
9. **Нативные оповещения**: рассылка пуш-уведомлений всем участникам топика по триггеру `@all` (скрытые упоминания).
10. **Shadow Auto-Registration**: Каждый пользователь автоматически вносится в базу данных при первом контакте через `ManagementService.ensure_user_registered`.
11. **Централизованное управление сущностями**: Полная консолидация бизнес-логики (мутации и регистрация) в `ManagementService` обеспечивает абсолютную целостность данных и SoC. Поддерживается **умный парсинг имен** с пробелами и отчествами.
12. **Протокол подтверждения (Safety First)**: Все деструктивные действия (удаление пользователей, групп, отзыв ролей) теперь требуют явного подтверждения. Это исключает потерю данных при случайных нажатиях.
13. **Типизированная навигация**: Централизованный навигатор теперь использует явные сигнатуры вызовов клавиатур, что гарантирует стабильность интерфейса и отсутствие ошибок аргументов.
14. **Автоматическое тестирование (QA)**: Полноценное покрытие тестами (`pytest`) всех слоев приложения: БД, бизнес-логика и хендлеры. Добавлены интеграционные тесты для проверки целостности шаблонов и каскадных удалений.
15. **Бронированная БД**: Усиленная схема с Foreign Keys на всех критических узлах и оптимизированные индексы для мгновенного доступа к правам в рантайме.

## 🛠 Troubleshooting

| Проблема | Причина | Решение |
| :--- | :--- | :--- |
| **RuntimeError: SQLite не поддерживает FK** | Библиотека SQLite в системе устарела или собрана без поддержки связей | Обновите Python/SQLite или смените хостинг. Бот не запустится без гарантии целостности данных |
| **sqlite3.OperationalError: database is locked** | Конкурентная запись в БД | Проверьте, что в `connection.py` включён WAL-режим |
| **Кнопка "Закрыть" не работает** | Потерян `last_menu_id` в FSM | Убедитесь, что используется `state.set_state(None)` |
| **Бот не удаляет сообщения в группе** | Недостаток прав | Бот должен быть администратором с правом «Удаление сообщений» |
| **Топик удалён в Telegram, но висит в БД** | Несовпадение жизненного цикла | Используйте кнопку «Удалить из БД» в меню топиков администратора |
| **Новая БД не работает** | Старый `bot.db` с устаревшей схемой | Удалите файл `database/bot.db` и перезапустите бота — схема пересоздастся автоматически |
| **OperationalError: no such table: topics** | Устаревшие запросы к БД | Таблица была переименована в `topic_names`. Убедитесь, что код обращается к актуальной схеме |
| **TypeError в клавиатурах** | Несоответствие аргументов | После рефакторинга многие клавиатуры (например, `main_admin_kb`) вызываются без аргументов. Проверьте сигнатуры в `UIService` |

## Настройка и запуск
1. Переименуйте `.env.example` в `.env` и заполните переменные (`BOT_TOKEN`, `ADMIN_ID`, `GROUP_ID`).
2. Используйте **Python 3.11**.
3. Создайте виртуальное окружение: `python -m venv venv`.
4. Активируйте окружение:
   - Windows: `.\venv\Scripts\activate`
   - Linux/macOS: `source venv/bin/activate`
5. Установите зависимости: `pip install -r requirements.txt`.
6. Запустите тесты: `pytest`.
7. Запустите бота: `python main.py`.
